# マルチエージェントのオーケストレーション

`/orchestrate` ワークフローで専門エージェントを連携させ、SaaS プラットフォーム全体を構築しましょう。

## 概要

ほとんどの AI とのやり取りは 1 対 1 です。`orchestrator` エージェントは**プロジェクトマネージャー**として振る舞うことで、これを変えます。
大きなリクエスト（例：「SaaS の MVP を作る」）をタスクに分解し、その仕事に最適な専門家へ割り当てます。

## プロセス

<StepList>
  <Step number={1} title="プロジェクトの概要">
    高レベルの目標を説明します。orchestrator がヒアリングして要件を明確にします。

    ```bash
    /orchestrate Build a "Course Platform" MVP with:
    - User Authentication (NextAuth)
    - Stripe Payments
    - Video Hosting (Mux)
    - Dashboard for Instructors
    ```
  </Step>
  <Step number={2} title="フェーズ 1：計画">
    `project-planner` エージェントが詳細なロードマップ（プロジェクトルートの `{task-slug}.md`）を作成します。

    <div className="bg-zinc-50 dark:bg-zinc-900 p-3 rounded-lg text-xs font-mono border-l-2 border-blue-500 mt-2">
      ...<br/>
      ### 2.1 Database Schema<br/>
      - Users (id, email, role)<br/>
      - Courses (id, title, price)<br/>
      ...
    </div>
  </Step>
  <Step number={3} title="フェーズ 2：実行">
    各エージェントが並行して作業します。Orchestrator は進捗を監視し、依存関係のブロックを解消します。
  </Step>
</StepList>

## 実践のヒント

<ProTips>
  <Tip title="小さく始める">
    一度に Facebook を作ろうとしないこと。まず MVP をオーケストレーションし、それから `/enhance` で反復しましょう。
  </Tip>
  <Tip title="認証を定義する">
    ユーザーがどうログインするかを必ず指定しましょう（例：「NextAuth と GitHub を使う」）。これは重要なアーキテクチャ上の判断です。
  </Tip>
</ProTips>
