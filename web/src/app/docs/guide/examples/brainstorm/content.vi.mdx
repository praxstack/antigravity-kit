# Động não có cấu trúc

Khám phá nhiều hướng tiếp cận trước khi viết code với quy trình `/brainstorm`.

## Tổng quan

Lệnh `/brainstorm` kích hoạt một **chế độ phát ý tưởng có cấu trúc**.
Thay vì lao thẳng vào code, tác tử khám phá ít nhất 3 hướng tiếp cận khác nhau,
phân tích ưu nhược điểm và giúp bạn ra quyết định sáng suốt.

<FeatureGrid>
  <Feature title="Không code" description="Chỉ ý tưởng" />
  <Feature title="3+ phương án" description="Luôn so sánh" />
  <Feature title="Ưu/Nhược" description="Đánh đổi thẳng thắn" />
  <Feature title="Bạn quyết" description="Lựa chọn cuối" />
</FeatureGrid>

## Quy trình

<StepList>
  <Step number={1} title="Mô tả thách thức của bạn">
    Cho tác tử biết bạn muốn khám phá gì. Hãy nêu rõ các ràng buộc.

    ```bash
    /brainstorm authentication system for my SaaS app
    ```
  </Step>
  <Step number={2} title="Xem lại các phương án">
    Tác tử trình bày 3+ hướng tiếp cận kèm ưu điểm, nhược điểm và mức công sức.
  </Step>
  <Step number={3} title="Ra quyết định">
    Chọn một phương án, rồi dùng `/enhance` hoặc `/create` để triển khai.
  </Step>
</StepList>
