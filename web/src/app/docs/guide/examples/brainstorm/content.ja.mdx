# 構造化ブレインストーミング

`/brainstorm` ワークフローで、コードを書く前に複数のアプローチを検討しましょう。

## 概要

`/brainstorm` コマンドは**構造化されたアイデア出しモード**を起動します。
いきなりコードに飛びつくのではなく、エージェントが少なくとも 3 種類の異なるアプローチを検討し、
それぞれの長所と短所を分析して、納得のいく意思決定を後押しします。

<FeatureGrid>
  <Feature title="コードなし" description="アイデアだけ" />
  <Feature title="3つ以上の選択肢" description="常に比較" />
  <Feature title="長所/短所" description="率直なトレードオフ" />
  <Feature title="決めるのはあなた" description="最終的な選択" />
</FeatureGrid>

## プロセス

<StepList>
  <Step number={1} title="課題を説明する">
    何を検討したいかをエージェントに伝えます。制約条件は具体的に。

    ```bash
    /brainstorm authentication system for my SaaS app
    ```
  </Step>
  <Step number={2} title="選択肢を確認する">
    エージェントが 3 つ以上のアプローチを、長所・短所・工数の目安とともに提示します。
  </Step>
  <Step number={3} title="意思決定する">
    1 つを選び、`/enhance` または `/create` で実装します。
  </Step>
</StepList>
