# 结构化头脑风暴

使用 `/brainstorm` 工作流，在动手写代码之前先探索多种方案。

## 概述

`/brainstorm` 命令会启动一种**结构化的构思模式**。
与其直接动手写代码，智能体会探索至少 3 种不同的方案，
分析各自的优缺点，帮助你做出明智的决策。

<FeatureGrid>
  <Feature title="不写代码" description="只谈想法" />
  <Feature title="3+ 个方案" description="始终对比" />
  <Feature title="优点/缺点" description="坦诚权衡" />
  <Feature title="由你决定" description="最终选择" />
</FeatureGrid>

## 流程

<StepList>
  <Step number={1} title="描述你的挑战">
    告诉智能体你想探索什么。把约束条件说清楚。

    ```bash
    /brainstorm authentication system for my SaaS app
    ```
  </Step>
  <Step number={2} title="审阅方案">
    智能体会给出 3 个以上的方案，并附上优点、缺点和工作量评估。
  </Step>
  <Step number={3} title="做出决策">
    选定一个方案，然后用 `/enhance` 或 `/create` 来实现。
  </Step>
</StepList>
